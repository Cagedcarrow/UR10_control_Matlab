3. 代码生成模式

MATLAB Coder 将符号表达式转换为 C++ 时遵循严格的模式：

阶段 1: 三角函数预计算

// 仅计算一次所有需要的三角函数
t2 = std::cos(in1[1]);
t3 = std::cos(in1[2]);
t4 = std::cos(in1[3]);
t7 = std::sin(in1[1]);
t8 = std::sin(in1[2]);
// 组合角度
t12 = in1[1] + in1[2];
t13 = std::cos(t12);
t15 = std::sin(t12);

阶段 2: 常量预计算与缓存

// 数值常量来自系统辨识
// 常用乘积被缓存以复用
gravity_current_tmp = t2 * t4;        // cos(q1)*cos(q3)
b_gravity_current_tmp = t3 * t4;     // cos(q2)*cos(q3)

阶段 3: 单表达式直接赋值

// 每个输出元素一个巨大的嵌套表达式
// 无循环、无递归、无分支
gravity_current[1] =
    ((((t2 * -5.5890515155147762 + t7 * 0.0302563171838271)
     - t13 * 2.6108363476259209 + t15 * 0.016866164608139979)
     + ((t17 * 0.0058448919007984988 + t19 * 0.19776029026325251)
     - t20 * 0.00339637375989025 - t21 * 0.014193086854840451)) + ... ;
为什么这样做更快: 传统递归 Newton-Euler 算法每次调用都需要 6 层递归遍历运动链，每层包含矩阵乘法和坐标变换。而闭式展开把所有计算展开为单一的顺序执行路径——零分支、零循环、零递归、零动态内存分配。CPU 的指令预取和流水线可以完美优化。

完整力矩公式

$$\boxed{\tau = \underbrace{K \cdot i_{arm}(q, \dot{q}, \ddot{q})}_{\text{本体动力学 (电流域)}} + \underbrace{K \cdot i_{friction}(\dot{q})}_{\text{摩擦 (电流域)}} + \underbrace{Y_L(q, \dot{q}, \ddot{q}) \cdot \pi_{load}}_{\text{负载 (力矩域)}}}$$

其中 $K = \text{diag}(K_1, \dots, K_6)$ 为电机增益矩阵。

5. Sigmoid 非线性摩擦模型

每个关节的摩擦电流公式：

$$i_{friction,j}(\dot{q}_j) = f_{v,j} \cdot \dot{q}_j + f_{o,j} + \frac{f_{c,j}}{1 + e^{-\alpha_j(\dot{q}_j + n_j)}}$$
参数物理意义J1J2J3J4J5J6
$f_v$粘性摩擦系数 [A/(rad/s)]1.0640.9940.6800.3160.2240.236
$f_o$偏移项 [A]−1.0070.956−0.812−0.177−0.192−0.245
$f_c$库仑摩擦幅值 [A]2.051−2.4021.6480.4690.4760.598
$\alpha$Sigmoid 斜率 [s/rad]7.947−59.95419.825134.898331.442459.193
$n$速度偏移 [rad/s]−0.0185−0.0020−0.0053−0.0186−0.0118−0.0130
Sigmoid vs 经典摩擦: 经典库仑摩擦在零速处存在 $\text{sign}(\dot{q})$ 不连续性，导致数值仿真抖动。Sigmoid 函数 $\frac{1}{1+e^{-\alpha(\dot{q}+n)}}$ 平滑逼近符号函数，参数 $\alpha$ 控制过渡陡峭度，$n$ 调整零速交叉点。

6. 电机增益矩阵

定义在 ur10_inverse_dynamics_solver.h 中：

static const double K1 = 13.5841;  // J1: shoulder_pan  [Nm/A]
static const double K2 = 14.2959;  // J2: shoulder_lift [Nm/A]
static const double K3 = 11.3716;  // J3: elbow         [Nm/A]
static const double K4 = 11.2408;  // J4: wrist_1       [Nm/A]
static const double K5 = 11.7681;  // J5: wrist_2       [Nm/A]
static const double K6 = 11.7682;  // J6: wrist_3       [Nm/A]

转换为对角矩阵：K = diag(K1, K2, K3, K4, K5, K6)，所有电流转换为力矩：$\tau = K \cdot i$。

8. Pluginlib 插件注册

两个求解器均通过 ROS2 pluginlib 注册，支持运行时动态切换：

// ur10_inverse_dynamics_solver.cpp 末尾
#include <pluginlib/class_list_macros.hpp>
PLUGINLIB_EXPORT_CLASS(ur10::InverseDynamicsSolverUR10,
                       inverse_dynamics_solver::InverseDynamicsSolver)

// kdl_inverse_dynamics_solver.cpp 末尾
PLUGINLIB_EXPORT_CLASS(inverse_dynamics_solver::InverseDynamicsSolverKDL,
                       inverse_dynamics_solver::InverseDynamicsSolver)

运行时加载：

pluginlib::ClassLoader<InverseDynamicsSolver> loader(
    "inverse_dynamics_solver", "inverse_dynamics_solver::InverseDynamicsSolver");
auto solver = loader.createUniqueInstance("ur10/InverseDynamicsSolverUR10");
solver->initialize(node, link_names);
auto torques = solver->getTorques(q, dq, ddq);  // 前馈力矩

9. 抽象基类接口

class InverseDynamicsSolver {
public:
    // 初始化: 从 ROS2 参数加载 mass, COM, inertia
    virtual void initialize(NodeSharedPtr node, vector<string> link_names);

    // === 纯虚函数 (子类必须实现) ===
    virtual MatrixXd getInertiaMatrix(VectorXd q) const = 0;
    virtual VectorXd getCoriolisVector(VectorXd q, VectorXd dq) const = 0;
    virtual VectorXd getGravityVector(VectorXd q) const = 0;
    virtual VectorXd getFrictionVector(VectorXd dq) const = 0;
    virtual tuple<MatrixXd,VectorXd,VectorXd>
        getDynamicParameters(VectorXd q, VectorXd dq) const = 0;

    // === 默认实现 (可重写) ===
    virtual VectorXd getTorques(VectorXd q, VectorXd dq, VectorXd ddq) const {
        return H*qdd + C*dq + g;  // 默认不含摩擦
    }

protected:
    array<double,10> inertial_parameters_;  // [mass, com_x, com_y, com_z,
                                             //  Ixx, Ixy, Ixz, Iyy, Iyz, Izz]
};
