电流模型辨识 (Current Model Identification)

基于实测 CSV 数据，通过 Simulink 管道验证线性电流估计模型，输出逐关节误差指标 (RMSE/MAE/MaxAbs)

1. 整体介绍

1.1 在机器人系统中的作用

电流模型辨识是连接理论动力学和实际电机控制的桥梁。UR10 驱动器的控制回路以电流为单位（而非力矩），因此需要精确的电流模型来实现：

  • 前馈电流控制：将期望运动的动力学需求（惯性 + 科里奥利 + 重力 + 摩擦）转换为参考电流 $i_{\text{ref}}$
  • 模型验证：对比简化线性模型和完整非线性模型的精度差异，评估辨识收益
  • 参数标定：通过批量评估管道，系统化地确定各关节的电流-运动映射关系
  • C++ 求解器基准：MATLAB 侧的线性模型为 C++ 逆动力学求解器中的高精度辨识模型提供了基准对比

1.2 输入与输出

符号名称维度来源
$q(t)$实测关节位置$N \times 6$CSV: Act_q0 ~ Act_q5
$\dot{q}(t)$实测关节速度$N \times 6$CSV: Act_qd0 ~ Act_qd5
$I_{\text{meas}}(t)$实测电机电流$N \times 6$CSV: Act_I0 ~ Act_I5
$\ddot{q}(t)$数值加速度$N \times 6$$\ddot{q}(t) = \Delta \dot{q}(t) / \Delta t$

输出：

  • 模型参数: $(b, k_q, k_{\dot{q}}, k_{\ddot{q}}) \in \mathbb{R}^{6 \times 4}$
  • 误差指标: RMSE, MAE, MaxAbs (逐关节)
  • 评估报告: summary.csv + overall_report.md

1.3 数学模型

简化线性模型 (MATLAB Simulink 中使用的版本)

$$\boxed{i_{\text{est},j} = b_j + k_{q,j} \cdot q_j + k_{\dot{q},j} \cdot \dot{q}_j + k_{\ddot{q},j} \cdot \ddot{q}_j}, \quad j = 1,\dots,6$$

其中 $\odot$ 表示逐元素乘积（每个关节独立建模，共 24 个参数）。

力矩通过电机增益转换：

$$\tau_{\text{est},j} = K_j \cdot i_{\text{est},j}$$
模型层级说明: 此线性模型是简化近似版。完整的逆动力学模型（C++ 侧）使用非线性 Sigmoid 摩擦、$6 \times 6$ 耦合惯性矩阵、科里奥利向量和重力向量的联合计算。线性模型忽略关节之间的动力学耦合，仅作为快速原型和模型复杂度的下限基准。

误差指标定义

对于每个关节 $j$，在 $N$ 个时间采样点上：

$$\text{RMSE}_j = \sqrt{\frac{1}{N}\sum_{t=1}^{N}\left(i_{\text{est},j}(t) - i_{\text{meas},j}(t)\right)^2}$$ $$\text{MAE}_j = \frac{1}{N}\sum_{t=1}^{N}|i_{\text{est},j}(t) - i_{\text{meas},j}(t)|$$ $$\text{MaxAbs}_j = \max_{t=1}^{N}|i_{\text{est},j}(t) - i_{\text{meas},j}(t)|$$

1.5 典型应用场景

  • 模型选型：快速对比不同模型复杂度（线性 → Sigmoid 摩擦 → 完整逆动力学）的精度收益
  • 数据采集后处理：批量评估管道自动扫描目录树中的所有 session_data.csv 文件，适用于机器人长时间运行后的数据批处理
  • 硬件健康监控：通过跟踪单关节误差趋势，检测减速器磨损（摩擦增加 → 残差增大）

2.2 函数详解

estimate_current_simple.m — 核心估计函数

function i_est = estimate_current_simple(q, dq, ddq, kq, kdq, kddq, b)
    q = q(:); dq = dq(:); ddq = ddq(:);          % 确保列向量
    kq = kq(:); kdq = kdq(:); kddq = kddq(:); b = b(:);
    i_est = b + kq .* q + kdq .* dq + kddq .* ddq;  % ★ 核心公式
end
设计要点: 函数签名中参数 $(b, k_q, k_{\dot{q}}, k_{\ddot{q}})$ 显式传入而非硬编码，使得该函数在 Simulink 中可参数化——修改 base workspace 中的变量即可改变模型行为，无需重建 Simulink 模型。这一设计支持参数扫描（遍历不同参数组合评估误差）。

evaluate_current_estimation_batch.m — 批量评估管道

function result = evaluate_current_estimation_batch(dataRoot, cfg, opts)
    % 1. 递归查找所有 session_data.csv
    csvFiles = dir(fullfile(dataRoot, '**/session_data.csv'));

    % 2. 加载 Simulink 模型并配置
    mdl = load_system(cfg.simModelName);

    % 3. 对每个文件执行 evaluateOneFile
    for i = 1:length(csvFiles)
        r = evaluateOneFile(mdl, cfg, fullfile(csvFiles(i).folder, csvFiles(i).name), ...);
        perFileMetrics{i} = r;    % 存储指标
    end

    % 4. 汇总统计
    overall = aggregateMetrics(perFileMetrics);
    % 5. 输出 summary.csv + overall_report.md
end

evaluateOneFile — 单文件评估

完整数据流：

  1. 读取 CSV: 提取 $q$, $\dot{q}$, $I_{\text{meas}}$ 矩阵 ($N \times 6$)
  2. 构建时间: $t = (0:N-1) \cdot \Delta t$
  3. 数值加速度: $\ddot{q} = [\text{zeros}(1,6); \text{diff}(\dot{q}) / \Delta t]$
  4. 配置 Simulink 输入: 创建 Simulink.SimulationInput，设置 5 个 From Workspace 变量
    • q_cmd_ts — 关节命令 (timeseries)
    • q_ref_ts — 实测位置参考
    • dq_ref_ts — 实测速度参考
    • tau_csv_ref_ts — CSV 中 tau_estimated 参考
    • tau_from_i_ref_ts — 从实测电流反算的力矩参考
  5. 运行仿真: sim(mdl, simIn)
  6. 提取结果: 从 yout 的第 4 个 Outport 取 $i_{\text{est}}$
  7. 计算误差: $e = i_{\text{est}} - i_{\text{meas}}$，逐关节计算 RMSE/MAE/MaxAbs
  8. 可选绘图: *_current_compare.png（最多 maxPlots 个文件）

参数规律: 注意到 $k_q, k_{\dot{q}}, k_{\ddot{q}}$ 在 J1-J3（基座/大臂/肘部）上显著大于 J4-J6（腕部）。这反映了物理规律——基座端关节驱动更大的惯量，需要更大的电流增益。偏置 $b$ 全为零，说明在辨识时假设了无摩擦偏置或已被吸收到 Sigmoid 偏移 $f_o$ 中。

4. 创新点分析

4.1 与标准辨识方法的对比

方面标准方法（激励轨迹 + LS）本项目方法（CSV 回放 + Simulink）
数据采集设计专用激励轨迹（如傅里叶级数轨迹），一次实验利用日常作业的 CSV 记录，零额外采集成本
模型结构完整逆动力学（$H\ddot{q} + C\dot{q} + g + f$），参数多简化线性模型（4 参数/关节），共 24 参数
参数估计方法最小二乘 / 加权最小二乘 / 最大似然固定经验参数（首版），待添加自动拟合
验证方式交叉验证集 / 留一法全套 Simulink 仿真：$i_{\text{est}}$ vs $i_{\text{meas}}$ 时间序列对比
可扩展性新机器人需重新设计激励轨迹仅需 CSV 数据，批处理管道通用

4.2 工程亮点

  • 端到端自动化管道：evaluate_current_estimation_batch 实现了从 CSV 发现 → Simulink 仿真 → 误差统计 → 报告生成的全自动流程。用户只需指定 dataRoot 目录，无需手动干预
  • 参数分离设计：estimate_current_simple 将模型参数作为函数参数传入（而非硬编码），使得参数扫描和模型对比可以通过修改 workspace 变量实现，无需重新生成 Simulink 模型
  • 线性模型作为复杂度下限：4 参数/关节的线性模型故意简化，其 RMSE 作为基准来衡量 C++ 侧 30 参数/关节的 Sigmoid 摩擦 + 耦合动力学模型的改进幅度
  • 多层次评估指标：RMSE（大误差惩罚重）、MAE（平均误差）、MaxAbs（最差情况）三者互补，全面评估模型在不同工况下的表现
  • 加速度激励度指标：输出 ddq_rms 用以评估 CSV 数据的"信息含量"。低 ddq_rms 的数据段（如匀速运动）对惯性参数不敏感，辨识结果不可靠——在报告中标记这些案例有助于数据质量诊断

4.3 局限性

  • 忽略关节耦合：线性模型对每个关节独立建模，忽略了惯性矩阵的非对角线元素和科里奥利耦合项（如 J2 的运动影响 J3 的负载）
  • 首版参数为固定经验值：当前 $b, k_q, k_{\dot{q}}, k_{\ddot{q}}$ 为手工设定（在 init_virtual_ur10.m 中硬编码），未经过系统性的最小二乘拟合
  • 无分工况辨识：同一组参数用于所有工况（静止、低速、高速、重载），导致在某些极端工况下误差显著增大
  • 数值加速度噪声：$\ddot{q}$ 通过对离散 $\dot{q}$ 做一阶差分获得，放大了测量噪声。对于高频运动，加速度估计的噪声可能掩盖真实的动力学信号
  • 数据依赖：CSV 文件路径 test/data/05_10_173354/session_data.csv 在当前代码库中不存在（可能为外部数据），批处理管道在实际运行前需确保数据目录路径正确

4.4 未来可改进方向

  • 以 summary.csv 的 RMSE 为目标函数，对 $(b, k_q, k_{\dot{q}}, k_{\ddot{q}})$ 进行梯度下降或最小二乘拟合
  • 分工况拟合：分别用静止段（$\dot{q} \approx 0$）、匀速段（$\ddot{q} \approx 0$）、加速段的数据独立拟合，建立工况相关的参数表
  • 将线性模型替换为解耦的 Sigmoid 摩擦 + 对角惯性模型（保持每个关节独立但增加摩擦非线性），在复杂度和精度之间取得更好的平衡
  • 引入低通滤波或卡尔曼滤波来改进加速度估计，减少数值微分噪声
  • 添加统计检验（如残差自相关性检验），判断模型是否系统性地遗漏了某些动力学效应
