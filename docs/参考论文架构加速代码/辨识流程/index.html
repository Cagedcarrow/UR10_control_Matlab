3. 需要辨识的内容

3.1 线性电流模型 (MATLAB 项目中的硬编码值)

文件 test/virtual/utils/estimate_current_simple.m 定义了简单的线性电流模型：

$$i_{est,j} = b_j + kq_j \cdot q_j + kdq_j \cdot \dot{q}_j + kddq_j \cdot \ddot{q}_j$$

每个关节 4 个参数 (bias, kq, kdq, kddq)，共 24 个参数。当前值是硬编码的启发值：

参数J1J2J3J4J5J6
bias000000
kq0.120.100.100.060.050.05
kdq0.850.880.820.450.400.36
kddq0.160.180.160.080.070.06
注意: 这些硬编码值精度极低。评估报告 (RMSE 达到数千安培) 表明它们与真实 UR10 行为偏差极大，不能直接用于任何实际控制。如需使用此线性模型，必须重新辨识。但如果你的目标是前馈力矩控制，应该使用 capsule 中的完整动力学求解器 (已辨识好)，而非这个简化线性模型。

3.2 换用新机器人时的完整重辨识

如果换用非 UR10 的机械臂，需要重新走完整辨识流程：

4. 完整辨识流程

步骤 1: 激励轨迹设计与数据采集

激励轨迹要求：

  • 轨迹需要充分激励所有关节的动力学特性
  • 覆盖各关节的完整运动范围
  • 包含不同速度段 (低速摩擦主导 → 高速惯性主导)
  • 建议使用傅里叶级数轨迹 (周期性, 便于频域分析)

数据采集格式 (CSV)：

列名维度单位说明
Act_q0 ~ Act_q56rad关节位置 (从 UR 驱动编码器读取)
Act_qd0 ~ Act_qd56rad/s关节速度 (编码器微分)
Act_I0 ~ Act_I56A电机电流 (UR 实时数据端口)
Time (可选)1s时间戳; 缺失则假定 50Hz 均匀采样

采样率: 项目中硬编码为 50Hz (sampleTime = 0.02s)。建议使用 125Hz (UR10 实时端口最大速率) 以获得更好的高频动力学信息。

数据采集建议: 使用 UR10 的 Real-Time Data Exchange 接口 (端口 30003, 125Hz) 采集数据，而非 MATLAB 的 Robotics System Toolbox 连接器。RTDE 提供同步的时间戳和低延迟数据。

步骤 2: 数据预处理

% 读取 CSV
data = readtable('session_data.csv');
q   = data{:, {'Act_q0','Act_q1','Act_q2','Act_q3','Act_q4','Act_q5'}};
dq  = data{:, {'Act_qd0','Act_qd1','Act_qd2','Act_qd3','Act_qd4','Act_qd5'}};
i   = data{:, {'Act_I0','Act_I1','Act_I2','Act_I3','Act_I4','Act_I5'}};

% 数值微分求加速度 (或用低通滤波 + 中心差分)
dt = 0.02;  % 50Hz
dq_filt  = lowpass(dq, 10, 50);      % 10Hz 截止低通滤波
ddq_filt = gradient(dq_filt) / dt;    % 中心差分
ddq_filt = lowpass(ddq_filt, 10, 50); % 再次滤波

% 去除静止段 (摩擦辨识不需要零速数据)
is_moving = vecnorm(dq_filt, 2, 2) > 0.01;
q   = q(is_moving, :);
dq  = dq_filt(is_moving, :);
ddq = ddq_filt(is_moving, :);
i   = i(is_moving, :);

步骤 3a: 简单线性模型辨识 (最小二乘)

MATLAB 中的 estimate_current_simple.m 使用的是各关节独立的线性模型：

% 对每个关节独立做最小二乘
for j = 1:6
    % 构造回归矩阵: X = [ones, q_j, dq_j, ddq_j]
    X = [ones(n,1), q(:,j), dq(:,j), ddq(:,j)];
    % 最小二乘求解: coeff = X \ i_meas
    coeff = X \ i(:,j);
    bias(j) = coeff(1);
    kq(j)   = coeff(2);
    kdq(j)  = coeff(3);
    kddq(j) = coeff(4);
end
局限性: 这种线性模型完全忽略了关节耦合效应。例如关节 2 的运动会影响关节 3 的负载力矩 (科里奥利耦合)，但线性模型假设各关节独立。因此它仅适用于低速、小范围运动的粗略估计。对于轨迹跟踪前馈控制，必须使用 capsule 中的完整动力学模型。

步骤 3b: 完整动力学辨识 (高级)

如需对非 UR10 机械臂进行完整辨识 (复现 capsule 中的方法)：

% 1. Symbolic Math Toolbox 推导 Lagrange 方程
syms q1 q2 q3 q4 q5 q6 real           % 关节角度
syms dq1 dq2 dq3 dq4 dq5 dq6 real      % 关节速度
syms ddq1 ddq2 ddq3 ddq4 ddq5 ddq6 real % 关节加速度
% ... 定义 DH 参数和惯性参数 ...

% 2. 推导 Lagrange 动力学
% L = T - V (动能 - 势能)
% τ = d/dt(∂L/∂dq) - ∂L/∂q
% 得到闭式符号表达式 H(q), C(q,dq), G(q)

% 3. 构造回归形式: τ = Y(q,dq,ddq) · π
% 其中 π 是待辨识的参数向量

% 4. 非线性优化辨识参数 π
options = optimoptions('lsqnonlin', 'Display', 'iter');
pi_opt = lsqnonlin(@(pi) costFunction(pi, Y, tau_meas), pi0, [], [], options);

步骤 4: 模型验证

项目中 evaluate_current_estimation_batch.m 提供了验证框架：

% 对验证集数据
for each session_data.csv:
    1. 读取 q_meas, dq_meas, i_meas
    2. 用辨识参数计算 i_est = f(q, dq, ddq_est)
    3. 计算误差:
       RMSE = sqrt(mean((i_est - i_meas).^2))
       MAE  = mean(abs(i_est - i_meas))
       MaxAbs = max(abs(i_est - i_meas))
    4. 生成诊断图 (时序对比 + 误差分布)

验收标准建议：

  • RMSE < 1A (对于典型 UR10 操作)
  • MAE < 0.5A
  • 误差应无系统性偏差 (残差白噪声)

步骤 5: MATLAB Coder 生成 C++ 并部署

完整动力学辨识完成后，用 MATLAB Coder 生成 C++ 代码并替换 capsule 中的对应文件：

% MATLAB Coder 配置
cfg = coder.config('lib');
cfg.TargetLang = 'C++';
cfg.GenerateExampleMain = 'DoNotGenerate';

% 对每个辨识得到的函数生成 C++
codegen -config cfg getInertiaCurrents -args {zeros(6,1)} -o getInertiaCurrents
codegen -config cfg getCoriolisCurrents -args {zeros(6,1),zeros(6,1)} -o getCoriolisCurrents
% ... 依次生成 8 个函数 ...

然后将生成的 .cpp 和 .h 文件复制到 ur10_inverse_dynamics_solver/ 并重新编译：

cd ros2_workspace
colcon build --packages-select ur10_inverse_dynamics_solver
colcon test --packages-select ur10_inverse_dynamics_solver  # 验证
