正运动学 (Forward Kinematics)

给定 6 个关节角度 q，通过 URDF 刚体树模型计算铲斗 TCP 的齐次变换矩阵 T（位姿）

1. 整体介绍

1.1 在机器人系统中的作用

正运动学是机器人系统的基础计算模块。它将关节空间（6 维角度向量）映射到笛卡尔工作空间（3D 位置 + 3D 姿态），是所有以下功能的前置条件：

  • Rviz 位姿可视化：在 MATLAB figure 中渲染机器人当前姿态
  • TCP 轨迹计算：给定关节角度序列，计算末端在空间中的运动路径
  • 逆运动学误差评估：验证 IK 求解结果的实际位姿与目标位姿的偏差
  • 碰撞检测前置：更新机器人各连杆的碰撞几何体位置

1.2 输入与输出

符号名称维度单位说明
$q$关节角度$6 \times 1$rad6 个旋转关节的位置
$T_{\text{TCP}}$齐次变换矩阵$4 \times 4$—$\begin{bmatrix} R & p \\ 0 & 1 \end{bmatrix}$, $R \in SO(3)$, $p \in \mathbb{R}^3$
posTCP 位置$3 \times 1$m从 $T$ 提取的平移分量 $(x,y,z)$
rpy姿态 (RPY)$3 \times 1$rad从 $T$ 提取的 XYZ 欧拉角 $(\phi,\theta,\psi)$

1.3 数学意义

正运动学的数学本质是齐次变换矩阵的链式乘积：

$$T_{\text{TCP}}(q) = T_0^1(q_1) \cdot T_1^2(q_2) \cdot T_2^3(q_3) \cdot T_3^4(q_4) \cdot T_4^5(q_5) \cdot T_5^6(q_6) \cdot T_6^{\text{TCP}}$$

其中每个 $T_i^{i+1}(q_i)$ 表示从连杆 $i$ 坐标系到连杆 $i+1$ 坐标系的齐次变换，是关节角度 $q_i$ 的函数。$T_6^{\text{TCP}}$ 是固定的末端工具偏移。

本项目未显式使用 DH 参数，而是通过 MATLAB 的 importrobot 从 URDF 文件自动解析所有坐标系变换。URDF 中的 <joint> 定义了各关节的旋转轴和原点偏移，MATLAB 内部将其转换为等价的齐次变换链。

  • 仿真后分析：run_virtual_trajectory_demo.m 仿真结束后计算完整 TCP 路径用于 3D 绘图

2.2 函数详解

fk_tcp_pose.m — 主 FK 包装器

function [T, pos, rpy] = fk_tcp_pose(robot, q)
    q = validate_joint_vector(q, []);        % 1. 验证 q 维度=6、值有限
    T = getTransform(robot, q, 'sensor_shovel_tcp');  % 2. 核心 FK 计算
    pos = tform2trvec(T);                          % 3. 提取位置 (x,y,z)
    rpy = tform2eul(T, 'XYZ');                     % 4. 提取欧拉角 (φ,θ,ψ)
end
调用链: validate_joint_vector → getTransform → tform2trvec / tform2eul
被调用者: ur10_gui_control_virtual.m (renderScene, updateTcpText), run_virtual_trajectory_demo.m (仿真后 TCP 计算)

rtfg_utils.m — 批量 FK 路径计算

function pos = getTcpPosition(robot, q)
    T = getTransform(robot, q, 'sensor_shovel_tcp');
    pos = tform2trvec(T);    % 仅返回位置，不提取姿态
end

function path = computeTcpPath(robot, qSeries)
    path = zeros(size(qSeries, 1), 3);
    for i = 1:size(qSeries, 1)
        path(i,:) = getTcpPosition(robot, qSeries(i,:));
    end
end
批量优化: computeTcpPath 采用逐行循环而非向量化方式，因为 getTransform 本身每次只能处理一个位姿。对于大型轨迹（140+ 采样点），这是明确的性能瓶颈，但保证了正确性。

build_ur10_virtual_robot.m — 机器人模型构建

function [robot, meta] = build_ur10_virtual_robot(opts)
    % 1. 预处理 xacro → 临时 URDF
    raw = fileread(opts.xacroPath);
    raw = regexprep(raw, '<xacro:arg[^>]*/>\s*', '');
    raw = strrep(raw, '$(arg mesh_root)', 'meshes');

    % 2. 导入为 rigidBodyTree
    robot = importrobot(tmpUrdf, 'DataFormat', 'row', 'MeshPath', meshDir);

    % 3. 去除 TCP 上的碰撞几何（避免自碰撞误检）
    robot = stripVirtualOnlyCollisions(robot);

    % 4. 验证恰好 6 个旋转关节
    meta.endEffector = 'sensor_shovel_tcp';
end

2.4 关节坐标系定义

UR10 的 6 个旋转关节通过 URDF <joint> 定义：

关节URDF 名称旋转轴父连杆子连杆原点偏移 (m)
J1ur10_shoulder_pan+Zbase_jizuour10_shoulder(0, 0, 0.1273)
J2ur10_shoulder_lift+Yur10_shoulderur10_upper_arm(0, 0.220941, 0)
J3ur10_elbow+Yur10_upper_armur10_forearm(-0.0000039, -0.1719, 0.612)
J4ur10_wrist_1+Yur10_forearmur10_wrist_1(-0.0000036, 0, 0.5723)
J5ur10_wrist_2-Zur10_wrist_1ur10_wrist_2(0.0000003, 0.1149, 0.0000003)
J6ur10_wrist_3+Yur10_wrist_2ur10_wrist_3(0, -0.0000003, 0.1157)
关节轴分布: UR10 的 6 个关节采用 Y-Y-Y 腕部结构（J4-J6），这是典型的球腕设计，使得腕部三个关节轴线交于一点，简化了逆运动学求解。

注意: J5 的旋转轴为 -Z（负方向），J1-J4 和 J6 为正方向。所有关节限制设置为 $[-2\pi, 2\pi]$（无硬限位）。

2.5 机器人模型来源

本项目统一使用 assembly/assembly.urdf.xacro 作为机器人模型的单一真相来源。该文件包含：

  • UR10 机械臂本体：6 个旋转关节 + 7 个连杆（含惯性和碰撞几何体）
  • 铲斗工具 (sensor_shovel)：通过固定关节 ur10-sensor_shovel 连接到手腕
  • TCP 参考点 (sensor_shovel_tcp)：固定偏移，用作末端执行器参考坐标系
  • 底座 (base_jizuo)：UR10 的安装底座

MATLAB 侧通过 importrobot(urdf, 'DataFormat', 'row') 加载，C++ 侧通过 kdl_parser::treeFromUrdfModel 加载。两者共享同一个 URDF 文件，保证了运动学模型的一致性。

4. 创新点分析

4.1 与标准方法的对比

方面标准方法（解析 DH）本项目方法（URDF 导入）
参数定义显式 DH 表：$a_i, d_i, \alpha_i, \theta_i$URDF <joint> 的 origin + axis
代码量需手写 6 个 $4 \times 4$ 矩阵的连乘函数一行 getTransform(robot, q, 'sensor_shovel_tcp')
可维护性模型修改需同步更新 DH 表和代码仅修改 URDF，所有消费者自动更新
跨平台一致性需手动保持 MATLAB/ROS 一致同一 URDF 被 MATLAB 和 ROS2 (KDL/MoveIt) 共享
额外功能无自动继承碰撞几何体用于碰撞检测；自动获取惯性参数用于动力学

4.2 工程亮点

  • 单一真相来源：URDF/Xacro 文件被 MATLAB 和 C++ ROS2 两侧共享，消除了多套参数之间的不一致风险。这是工业级机器人项目的关键实践
  • 批量 FK 模式：computeTcpPath 的逐点计算虽然简单，但作为 rtfg_utils.m 的公共接口被多个模块复用，避免了分散的重复实现
  • TCP 碰撞剥离：build_ur10_virtual_robot.m 主动移除 TCP 参考点的碰撞几何体，避免将工具坐标系误检为自碰撞。这是从实际调试中产生的针对性优化

4.3 局限性与改进方向

  • 性能：批量 FK (computeTcpPath) 使用 for 循环逐点调用 getTransform，对于数千个采样点的轨迹可能成为瓶颈。可考虑 MATLAB 的内部批处理 API 或 MEX 加速
  • 依赖：完全依赖 MATLAB Robotics System Toolbox，无法在 GNU Octave 等开源替代环境中运行
  • 封闭性：未暴露 DH 参数表，使得直接与教科书公式对照不方便，不利于教学场景
