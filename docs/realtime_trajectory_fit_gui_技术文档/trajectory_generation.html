4.3 三段式轨迹几何（YOZ 平面）

轨迹在 YOZ 平面内由三个连续段组成，对应挖掘动作的三个阶段：

Z (上) ^ | pArcEnd ●──────────────────● pLiftEnd ← 段2: 提升 (垂直上升, 90点) | | | 圆弧挖掘 | | (半径 R) | | | mudZ ─ ─ ● pDeepest | 泥面 | | | | pEntry ● | ← 段1: 圆弧 (140点) / | / | / θ = -30° | / | pStart ●──────────────────────+──────────→ Y (水平) 料斗左壁 ← 段0: 切入 (斜线下降, 30点) →

段 0 — 切入段（Approach）

  • 形状：从 pApproachStart 到 pEntry 的直线
  • 方向：[cos(θ), sin(θ)]，θ 为负表示向下倾斜
  • 点数：n0 = 30
  • 目的：将铲斗从上方引导至料斗内的泥面附近，避免与料斗壁碰撞

段 1 — 圆弧段（Arc / Scoop）

  • 形状：从 pEntry 到 pArcEnd 的圆弧
  • 半径：R = depth / (1 - cos(|θ|))，由穿透深度和切入角决定
  • 圆心：center = pEntry - R · [cos(θ - π/2), sin(θ - π/2)]
  • 扫描角度：从 θ - π/2 扫到 -π/2
  • 点数：nArc = 140
  • 目的：模拟铲斗从泥面切入后沿弧形路径挖掘并抬升

段 2 — 提升段（Lift）

  • 形状：从 pArcEnd 垂直上升到料斗顶部（Y 不变，Z 递增）
  • 长度：min(basinZMotionMax - pArcEndZ, max(0.06, 1.2 × R))，最短 6 cm
  • 点数：nLift = 90
  • 目的：将铲斗垂直提升出料斗，避免碰撞料斗壁

4.4 2D → 3D 提升

generate_trajectory_3d.m 将 YOZ 平面轨迹提升到恒定 X 平面：

% 每个 2D 点 [y, z] → 3D 点 [xPlane, y, z]
pts3d = [repmat(xPlane, size(pts2d,1), 1), pts2d];

% 切向量通过中心差分计算并归一化
tangent = normalize(diff3(pts3d) / stepSize);

轨迹点的 X 坐标全部等于 xPlane（由 YAML 配置或料斗原点 X 决定），使得整个轨迹位于一个平行于 YOZ 的平面内。

4.5 姿态应用

rtfg_utils('applyPoseToTrajectory', traj, pose) 将轨迹从料斗局部坐标系变换到机器人基坐标系：

% R = Rz(yaw) * Ry(pitch) * Rx(roll)  (ZYX 内旋)
% 位置变换:  ptsOut = R * ptsIn' + translation
% 向量变换:  vecOut = R * vecIn'  (仅旋转，不平移)

pose 来自 environment_runtime_config.yaml 中的 base_jizuo_to_block_with_basin_frame（典型值：x=-1.830m, yaw=180°），表示料斗在机器人前方 1.83 m 处，朝向与机器人相反。

4.6 目标变换构建

buildTrajectoryTargetPlan 为每个轨迹点构造 4×4 齐次变换矩阵，作为 IK 求解的目标位姿：

切入段与圆弧段（段0 + 段1）

  • X 轴：切线反方向（铲斗"背部"朝向运动方向）→ xAxis = -normalize(tangent)
  • Z 轴：世界 Z 轴 [0,0,1] 投影到 X 正交平面后归一化 → 尽可能保持向上
  • Y 轴：右手法则 yAxis = cross(zAxis, xAxis)

提升段（段2）

  • 恒定朝向：使用圆弧段最后的水平方向平均作为整个提升段的固定朝向
  • 避免提升过程中铲斗姿态摆动

圆弧→提升过渡

  • 使用 SLERP（球面线性插值） 在最后圆弧姿态和提升姿态之间平滑过渡
  • 过渡步数自适应（8-18 步），取决于角度差异
  • 混合函数使用 smoothstep：α = 3t² - 2t³
