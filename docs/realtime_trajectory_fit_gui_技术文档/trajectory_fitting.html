5. 轨迹拟合算法与计算速度

阻尼最小二乘 IK、四阶段权重调度、渐进式多种子策略、quintic blend 回放

5.1 问题定义

给定 N 个目标末端执行器位姿（4×4 齐次变换矩阵），求解一条在关节空间中连续、无碰撞、运动平滑的关节轨迹 q₁, q₂, …, qₙ。

核心约束：

  • 位置精度：TCP 位置误差 ≤ 3 cm
  • 碰撞安全：自碰撞、工具-本体、工具-料斗间距 ≥ 2 mm
  • 关节连续：相邻锚点间关节步长 < 3°（回放插值后 < 0.7°）
  • 关节限位：所有关节角在硬件限位范围内

5.2 阻尼最小二乘法（DLS）求解 IK

每个锚点的 IK 由 solveSinglePose() 求解，使用带自适应阻尼的高斯-牛顿迭代法。

5.2.1 姿态误差

\[ \mathbf{e} = \begin{bmatrix} \mathbf{p}_{\text{target}} - \mathbf{p}_{\text{current}} \\ \mathbf{R}_{\text{current}} \cdot \log(\mathbf{R}_{\text{current}}^T \mathbf{R}_{\text{target}}) \end{bmatrix} \in \mathbb{R}^6 \]

位置误差为世界坐标系下的欧几里得差（米）。旋转误差在 TCP 体坐标系下计算为旋量 log(R)（轴-角表示），再通过 R_current 变换回世界坐标系。这使得旋转误差在数值上等价于角速度误差（弧度），可直接用于 IK 求解。

5.2.2 加权误差与雅可比

\[ \mathbf{e}_w = \mathbf{W} \mathbf{e}, \quad \mathbf{J}_w = \mathbf{W} \mathbf{J} \]

其中 \(\mathbf{W} = \text{diag}(w_{tx}, w_{ty}, w_{tz}, w_{rx}, w_{ry}, w_{rz})\) 为对角权重矩阵。权重由四阶段调度决定（详见 5.3）。

5.2.3 数值雅可比

使用一阶前向差分计算 6×6 空间雅可比：

\[ \mathbf{J}_{:,i} = \frac{\text{poseError}(\mathbf{T}_0, \mathbf{T}(\mathbf{q} + \varepsilon \mathbf{e}_i))}{\varepsilon}, \quad \varepsilon = 10^{-6} \]

每次雅可比计算需要 6 次 KDL 正运动学调用（每次 ~0.02-0.05 ms），共 ~0.12-0.30 ms。

5.2.4 DLS 更新

\[ \mathbf{H} = \mathbf{J}_w^T \mathbf{J}_w + \lambda \mathbf{I}_{6 \times 6} \] \[ \mathbf{g} = \mathbf{J}_w^T \mathbf{e}_w \] \[ \Delta\mathbf{q} = \mathbf{H}^{-1} \mathbf{g} \quad \text{(LDLT 分解)} \]

步长限制：如果 \(\|\Delta\mathbf{q}\| > 0.45\) rad，按比例缩至该上限。更新后的关节角经过 clampToLimits（限位裁切）和 alignToReference（处理 J5/J6 ±2π 跳变）。

5.2.5 自适应阻尼

if (pos_err > 0.1 || rot_err > 0.1) {
    // 远离目标 → 强阻尼防止发散
    lambda = clamp(5e-3 * pos_err / 0.05, 1e-3, 0.1);
} else {
    // 接近目标 → 弱阻尼允许精细调整
    lambda = 5e-4 + 5e-3 * pos_err / 0.05;
}

阻尼 λ 平滑过渡（无二元切换），基于残差大小自适应。远离目标时大阻尼（~0.1）防止步长过大导致震荡；接近目标时小阻尼（~5e-4）允许快速收敛。

5.2.6 停滞检测

if (err_norm >= prev_err_norm * 0.99) {   // 误差改善 < 1%
    stagnate_count++;
    if (stagnate_count >= 8) break;       // 连续 8 步无显著改善 → 终止
} else {
    stagnate_count = 0;
}

5.2.7 收敛条件

if (pos_err <= 0.03 && rot_err <= orient_limit) converged = true;

位置容限固定为 3 cm；旋转容限由权重调度动态设置。

5.5 延迟碰撞验证

碰撞检测（FCL 距离查询）是计算密集型操作（~15 ms/次）。try_candidate lambda 实现了两层过滤：

auto try_candidate = [&](const CandidateInfo& cand, double orient_limit) {
    if (!cand.valid) return;  // IK 失败 → 跳过

    // 始终跟踪最佳备选（无需碰撞检查）
    updateBestFallback(cand);

    // IK 容限门控：只有 IK 收敛的候选才检查碰撞
    if (cand.pos_err > 0.03 || cand.rot_err > orient_limit) return;

    // 延迟碰撞检查（仅此处调用一次）
    auto collision = evaluateConfiguration(robot, basin_boxes, cfg, cand.q);
    double clearance = min({collision.min_self, collision.min_tool_body, collision.min_tool_basin});

    if (clearance < 2mm) return;  // 碰撞 → 跳过

    // 碰撞安全的候选 → 按代价/间距排序
    updateBestSafe(cand, clearance);
};

效果：碰撞检查次数从 ~100/姿态（每种子都检查）降至 ~1-3/姿态（仅 IK 可行候选），节省 6.2×。

5.6 回放生成 — Quintic Blend

锚点之间的回放关节轨迹使用 quintic blend（五次样条混合） 插值：

\[ h(t) = 6t^5 - 15t^4 + 10t^3, \quad t \in [0, 1] \] \[ \mathbf{q}(t) = \mathbf{q}_a + h(t) \cdot \text{wrapJointDelta}(\mathbf{q}_b - \mathbf{q}_a) \]

关键性质：

  • \(h(0) = 0, h(1) = 1\) — 边界插值正确
  • \(h'(0) = h'(1) = 0\) — 零初末速度（C¹ 连续）
  • \(h''(0) = h''(1) = 0\) — 零初末加速度（C² 连续）

子段数自适应：每对锚点间回放点数基于三种步长约束的最大值：

nseg = 1 + max(2,
    ceil(max_joint_step / 0.7 deg),    // 关节步长 ≤ 0.7°
    ceil(max_pos_step / 3.0 mm),        // TCP 步长 ≤ 3 mm
    ceil(max_rot_step / 0.3 deg));      // 旋转步长 ≤ 0.3°
nseg = clamp(nseg, 4, 32);             // 最少 4 段，最多 32 段

5.7 稀疏碰撞审计

回放轨迹的碰撞审计采用 stride=10 的稀疏采样：

const int collision_stride = 10;
for (int i = 0; i < playback_q.rows(); ++i) {
    bool do_check = (i % collision_stride == 0) || (i == 0) || (i == playback_q.rows() - 1);
    if (!do_check) continue;
    auto collision = evaluateConfiguration(...);
    updateGlobalMinima(collision, i);
}

设计依据：相邻回放点关节步长 < 0.7°，碰撞状态在两点间不可能突变。锚点已在求解阶段验证无碰撞。~120 次采样即可覆盖全部 ~1185 个回放点，节省 ~90% 检查时间。

5.8 连续性代价

当多个候选均无碰撞时，使用连续性代价作为平局打破器：

\[ \text{cost} = \|\Delta\mathbf{q}_{\text{pos}}\| + 0.65 \cdot \|\Delta\mathbf{q}_{\text{vel}}\| \]

惩罚关节位置突变（主项）和速度突变（加权 0.65），选择运动最平滑的候选。
