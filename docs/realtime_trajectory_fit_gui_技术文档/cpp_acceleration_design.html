MEX 编译流程、C++ 源文件清单、节点关系云图

2.1 为什么用 C++/MEX

MATLAB 的 inverseKinematics 函数虽然成熟稳定，但在大规模多种子求解场景下存在性能瓶颈：

  • 解释器开销：每次调用 ikSolver(eeName, tform, weights, qSeed) 需要 MATLAB 解释器解析、分发、类型检查，单个调用开销约 10-15ms
  • 数据序列化：MATLAB 与底层 C++ 求解器之间需要 mxArray ↔ 内部格式转换
  • 多种子求解：每姿态最多尝试 ~120 个种子 × 4 组权重 = ~480 次 IK 调用，累积开销极大

MEX (MATLAB Executable) 允许将 C/C++ 代码编译为 MATLAB 可直接调用的共享库（.mexa64），消除解释器开销，将所有计算保留在单一编译单元内。

2.2 编译流程

build_mex.m 使用 g++（C++17，-O3 优化）编译 6 个 .cpp 源文件，链接 16 个外部库：

% build_mex.m 核心配置
args = {'-v', '-largeArrayDims', ...
    'CXXFLAGS=$CXXFLAGS -std=c++17 -O3', ...
    'LDFLAGS=$LDFLAGS -Wl,--disable-new-dtags ...'};

includeDirs = {'/usr/include/eigen3', ...
               '/opt/ros/humble/include', ...
               '/opt/ros/humble/include/kdl_parser', ...};

libs = {'-lorocos-kdl', '-lkdl_parser', '-lurdf', ...
        '-lfcl', '-lccd', '-loctomap', '-loctomath', ...};

mex(args{:}, srcFiles, '-output', 'rtfg_solver_mex');

RPATH 配置

MATLAB 的 MEX 加载器不会自动搜索 ROS 2 库路径。通过 --disable-new-dtags 强制使用 DT_RPATH（而非 DT_RUNPATH），使 RPATH 传播到所有传递依赖：

LDFLAGS=-Wl,--disable-new-dtags \
        -Wl,-rpath,/opt/ros/humble/lib \
        -Wl,-rpath,/opt/ros/humble/lib/x86_64-linux-gnu

2.4 模块职责与关键函数

mex_entry.cpp — MEX 网关

void mexFunction(int nlhs, mxArray* plhs[], int nrhs, const mxArray* prhs[]) {
    // 1. 解析 MATLAB struct → C++ 类型
    auto target_tforms = readTformStack(getField(in, "targetTforms"));
    auto segment_names = readCellstrVector(getField(in, "segmentNames"));
    // 2. 加载机器人模型
    RobotModel robot = loadRobotModel(scene_urdf, base_link, tip_link);
    // 3. 求解轨迹
    TrajectoryResult result = solveTrajectory(robot, basin_boxes, cfg,
        target_tforms, segment_names, current_q, initial_q);
    // 4. 封装返回 MATLAB struct
    plhs[0] = buildOutputStruct(result);
}

函数输入输出职责

ik_solver.cpp — IK 求解器

函数算法关键参数
solveSinglePose阻尼最小二乘法 (DLS)λ 自适应 [5e-4, 0.1], 迭代 ≤ 60, 停滞检测 8 步
buildSeedList3 基种子 × 9 包裹组合 (J5, J6 ±2π)去重容差 1e-9 rad
buildGlobalSeedList中位 + 96 均匀随机 (rng=42)关节限位内均匀分布

collision_checker.cpp — 碰撞检测

检测类型方法性能
自碰撞 (self)O(N²) 对距离查询，跳过相邻链节 (|Δidx| ≤ 1)~15 ms/次
工具-本体 (tool_body)工具链节 × 本体链节距离查询包含在上述
工具-料斗 (tool_basin)工具链节 × 料斗 Box 距离查询+ ~1 ms/box
关键优化：使用 thread_local 缓存 FCL CollisionObject，后续调用仅更新变换矩阵 (setTransform) 和 AABB，避免每次重建 BVH 树（~30ms → ~1ms 开销）。
