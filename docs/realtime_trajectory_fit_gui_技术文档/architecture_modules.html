1.2 Action-Dispatcher 模式

每个功能模块遵循统一的调用约定：

function varargout = module_name(action, varargin)
    switch action
        case 'action1'
            [out1, out2] = localFunction1(varargin{:});
        case 'action2'
            ...
    end
end

这种模式使得每个模块对外暴露一组具名操作，内部实现对调用者透明，便于独立测试和替换（如将 MATLAB 碰撞检测替换为 C++ FCL）。

1.5 数据流：state 结构体

整个应用共享一个 state 结构体，在所有模块间传递：

state
├── .robot           — rigidBodyTree 对象 (MATLAB) / KDL Chain (C++)
├── .trajParams      — 轨迹参数 (mudHeight, approachAngle, arcRadius, ...)
├── .pose            — 铲斗位姿 (x, y, z, rollDeg, pitchDeg, yawDeg)
├── .envGeom         — 料斗几何 (basin.innerXMin, floorZ, ...)
├── .traj            — 原始轨迹 (generate_trajectory_3d 输出)
├── .displayTraj     — 应用位姿后的轨迹
├── .previewQSeries  — 拟合后的回放关节序列
├── .previewMetrics  — 最大关节步长、旋转增量等统计
├── .collisionResults — 碰撞检测结果
├── .statusText      — 当前状态描述
├── .progress        — 进度 (0~1)
└── .runEnabled      — 是否允许播放

1.6 MEX 调用路径详解

当 rtfg_kinematics('trackTrajectory', state) 被调用时，MEX 快速路径的执行流程如下：

  1. 验证 MEX 存在：exist('rtfg_solver_mex', 'file') == 3
  2. 构建 inputStruct：
    • targetTforms：4×4×N 齐次变换矩阵（通过 buildTrajectoryTargetPlan 生成）
    • sceneUrdf / collisionUrdf：URDF 文件路径
    • currentQ / initialQ：当前关节角 / 初始关节角
    • segmentNames：每点的段标签（approach / arc / lift）
    • basinBoxes：料斗碰撞盒
  3. 调用 MEX：solverResult = rtfg_solver_mex(inputStruct)
  4. 提取结果：anchorQSeries、playbackQSeries、tcpPath、collisionResults、previewMetrics、progressEvents
