逆动力学 (Inverse Dynamics)

给定关节位置 q、速度 dq、加速度 ddq，计算维持该运动所需的关节力矩 τ

1. 整体介绍

1.1 在机器人系统中的作用

逆动力学是机器人力矩控制的基础。它将期望的运动轨迹（关节位置/速度/加速度）映射为所需的驱动力矩，在以下场景中至关重要：

  • 前馈力矩控制：在轨迹跟踪控制器中，将计算力矩作为前馈项，显著提高跟踪精度
  • 重力补偿：在拖动示教/零力控制中，补偿重力使机械臂感觉"失重"
  • 动力学参数辨识：回归矩阵形式支持在线辨识负载参数
  • 碰撞检测：$\tau_{\text{measured}} - \tau_{\text{computed}}$ 的残差可以检测外部碰撞
  • 能耗分析与优化：精确的力矩预测用于轨迹能耗评估

符号名称维度单位说明

求解器还能单独返回：

  • $H(q) \in \mathbb{R}^{6\times6}$ — 惯性矩阵（对称正定）
  • $C(q,\dot{q})\dot{q} \in \mathbb{R}^6$ — 科里奥利/离心力向量
  • $g(q) \in \mathbb{R}^6$ — 重力向量
  • $f(\dot{q}) \in \mathbb{R}^6$ — 摩擦力向量

1.3 数学模型

标准刚体逆动力学方程：

$$\tau = H(q)\ddot{q} + C(q,\dot{q})\dot{q} + g(q) + f(\dot{q})$$

本项目的核心创新是将模型表述为电流级（而非传统的力矩级），并采用双寄存器架构：

完整力矩公式

$$\boxed{\tau = K \cdot i_{\text{arm}}(q, \dot{q}, \ddot{q}) + K \cdot i_{\text{friction}}(\dot{q}) + Y_L(q, \dot{q}, \ddot{q}) \cdot \pi_{\text{load}}}$$

其中：

  • $K = \text{diag}(K_1, \dots, K_6)$ — 电机增益矩阵 (Nm/A)，将电流转换为力矩
  • $i_{\text{arm}}$ — 机械臂本体的电流贡献（惯性 + 科里奥利 + 重力）
  • $i_{\text{friction}}$ — Sigmoid 非线性摩擦电流
  • $Y_L$ — 负载回归矩阵 ($6 \times 19$)
  • $\pi_{\text{load}}$ — 19 维负载动力学系数

// 1. 惯性矩阵: H = K * B_arm + B_load
Eigen::MatrixXd getInertiaMatrix(const Vector6d& q) const {
    getInertiaCurrents(q, B);                             // arm: B(6×6) 电流矩阵
    getPayloadInertiaRegressionMatrix(q, inertial, BL);   // payload: BL(6×6)
    return K * B + BL;
}

// 2. 科里奥利向量: C = K * c_arm + YCL * π_load
Eigen::VectorXd getCoriolisVector(q, qd) const {
    getCoriolisCurrents(q, qd, coriolis_i);               // arm: 电流(6)
    getPayloadCoriolisRegressionMatrix(q, qd, YCL);       // payload: YCL(6×19)
    return K * coriolis_i + YCL * dynamicCoefficients();
}

// 3. 重力向量: g = K * g_arm + YGL * π_load
Eigen::VectorXd getGravityVector(q) const {
    getGravityCurrents(q, gravity_i);                     // arm: 电流(6)
    getPayloadGravityRegressionMatrix(q, YGL);            // payload: YGL(6×19)
    return K * gravity_i + YGL * dynamicCoefficients();
}

// 4. 摩擦力: f = K * i_friction(qd)
Eigen::VectorXd getFrictionVector(qd) const {
    return K * getFrictionCurrents_(qd);
}

// 5. 总力矩 (重写基类): τ = K*i_arm + K*i_friction + YL*π_load
Eigen::VectorXd getTorques(q, qd, qdd) const {
    getCurrents(q, qd, qdd, currents);                    // arm: 总电流(6)
    getPayloadRegressionMatrix(q, qd, qdd, YL);           // payload: YL(6×19)
    return K * currents + getFrictionVector(qd) + YL * dynamicCoefficients();
}

Sigmoid 非线性摩擦模型

每个关节 $j \in \{1,\dots,6\}$ 的摩擦电流：

$$i_{\text{friction},j}(\dot{q}_j) = f_{v,j} \cdot \dot{q}_j + f_{o,j} + \frac{f_{c,j}}{1 + e^{-\alpha_j(\dot{q}_j + n_j)}}$$
参数含义J1J2J3J4J5J6
$f_v$粘性摩擦1.0640.9940.6800.3160.2240.236
$f_o$偏移−1.0070.956−0.812−0.177−0.192−0.245
$f_c$库仑幅值2.051−2.4021.6480.4690.4760.598
$\alpha$斜率7.947−59.95419.825134.898331.442459.193
$n$偏移−0.0185−0.0020−0.0053−0.0186−0.0118−0.0130
Sigmoid 摩擦 vs 经典库仑+粘性: 经典库仑摩擦在零速处存在不连续性 ($\text{sign}(\dot{q})$)，在数值仿真中引起抖动。Sigmoid 通过平滑 S 形曲线 ($\frac{1}{1+e^{-\alpha(\dot{q}+n)}}$) 逼近符号函数，消除了不连续性，使得 Simulink 仿真和实时控制更加稳定。参数 $\alpha$ 控制过渡的陡峭程度，$n$ 调整零速交叉点。

19 维负载动力学系数

从 10 个标准惯性参数（质量 $m$、质心 $c = (c_x,c_y,c_z)$、惯性张量 $\mathbf{I}$）派生出的 19 个扩展参数：

$$\pi_{\text{load}} = [m,\; m c_x,\; m c_y,\; m c_z,\; I_{xx},\; m c_y^2,\; m c_z^2,\; I_{xy},\; m c_x c_y,\; I_{xz},\; m c_x c_z,\; I_{yy},\; m c_x^2,\; m c_z^2,\; I_{yz},\; m c_y c_z,\; I_{zz},\; m c_x^2,\; m c_y^2]^T$$

这 19 维参数是对完整负载动力学参数空间的最小参数化，包含了质量与惯性矩的所有线性独立组合。回归矩阵 $Y_L \in \mathbb{R}^{6 \times 19}$ 将参数向量线性地映射到关节力矩。

4. 创新点分析

4.1 与标准方法的对比

方面标准方法 (RBDL/Pinocchio)本项目方法 (电流级回归 + 双寄存器)
建模层级力矩级：直接计算 $\tau$电流级：$i = f(q,\dot{q},\ddot{q})$，然后 $\tau = K \cdot i$
参数来源CAD 模型 → 惯性参数理论值MATLAB 系统辨识 → 实验数据拟合参数
摩擦模型通常仅粘性 + 库仑（含符号函数不连续）Sigmoid 平滑摩擦（30 参数，避免数值抖动）
负载处理修改 URDF 后重新计算线性回归矩阵：$\tau_{\text{load}} = Y_L(q,\dot{q},\ddot{q}) \cdot \pi$，支持在线辨识
可扩展性新机器人 = 新 URDF + 重新编译新机器人 = MATLAB 重新辨识 + 重新生成 8 个文件

4.2 工程亮点

  • 电流级建模：在电机电流层面而非力矩层面建模，直接匹配 UR10 驱动的电流反馈环。这使得模型可以绕过力矩传感器的噪声和延迟，直接与底层电机控制回路对接
  • 双寄存器解耦架构：机械臂本体动力学（电流级）和负载动力学（力矩级）分离为两个独立的寄存器。机械臂本体的 8 个生成函数完全来自实验辨识（而非 CAD 理论值），负载部分通过线性回归矩阵支持任意末端工具的运行时补偿
  • MATLAB → C++ 自动代码生成管道：Symbolic Math Toolbox 的符号推导 → MATLAB Coder 的代码生成，保证了辨识模型到运行时求解器之间的零误差传递。合计约 25,000 行的自动生成代码通过单元测试验证了数值精度（$10^{-6}$ 容差）
  • pluginlib 多态架构：KDL 求解器（通用仿真）和 UR10 求解器（高精度真机）共享同一接口，上层应用无需修改代码即可切换求解器。这在开发和调试阶段非常实用——可以在仿真中用 KDL 快速验证，在真机上切换到 UR10 求解器
  • 19 维负载参数化：10 个标准惯性参数（$m$, COM, $\mathbf{I}$）被展开为 19 个线性独立组合，包含了所有质量/质心/惯性张量的交叉项。这使得回归矩阵 $Y_L$ 可以线性地表示负载对任意关节力矩的贡献，支持最小二乘在线辨识

4.3 局限性

  • 生成代码膨胀：8 个生成文件合计约 25,000 行，编译时间长。如果改用 Pinocchio 的运行时刚体算法 (ABA/RNEA)，代码量可减少 90% 以上
  • 辨识依赖：模型精度完全取决于辨识实验的质量。如果辨识数据未覆盖某些运动工况（如高速、大负载），模型在那些区域的预测会显著偏差
  • KDL 无摩擦：KDL 求解器缺失摩擦模型，限制了其在仿真中的保真度。在摩擦占主导的低速段（如精细装配），KDL 仿真与 UR10 实际行为偏差较大

4.4 未来可改进方向

  • 集成 Pinocchio 替代 KDL，获得更快的计算速度和自动微分支持
  • 分工况（静止/低速/高速）辨识多组摩擦参数，运行时根据速度范围动态插值
  • 将 19 维负载回归扩展为支持在线递推最小二乘 (RLS) 的实时负载辨识器
  • 添加关节柔性（谐波减速器）模型，提高力矩预测在换向瞬间的精度
