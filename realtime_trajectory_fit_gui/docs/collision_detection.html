碰撞检测逻辑

概述

碰撞检测基于 FCL (Flexible Collision Library)，对 UR10 挖掘机配置进行三类碰撞检测：

类型检测对象优先级说明
自碰撞 (self)机器人各连杆之间低UR10 本体连杆相互碰撞
工具-本体 (tool_body)铲斗 vs 机器人本体中传感器铲斗碰到 UR10 手臂
工具-料斗 (tool_basin)铲斗 vs 料斗壁高铲斗碰到料斗内壁/底面

碰撞模型加载

碰撞几何体从 URDF 文件的 <collision> 标签加载：

  • Box：长方体 → fcl::Boxd
  • Cylinder：圆柱体 → fcl::Cylinderd
  • Mesh：STL 网格 → fcl::BVHModel<OBBRSSd>（支持二进制和 ASCII STL）

工具连杆标记：sensor_shovel 和 sensor_shovel_tcp。

FCL 检测流程

evaluateConfiguration(robot, basin_boxes, cfg, q):
  1. 计算正运动学 (FK) → 所有连杆的世界位姿
  2. 为每个碰撞几何体创建/更新 FCL CollisionObject
     - 首次调用：创建对象并缓存 (thread_local)
     - 后续调用：仅更新变换 (setTransform) + AABB
  3. 自碰撞 & 工具-本体检测：
     对所有连杆对 (i, j), i < j, |chain_index_i - chain_index_j| > 1:
       fcl::distance(obj_i, obj_j) → 最小距离
       根据工具标记分类为 self 或 tool_body
  4. 工具-料斗检测：
     对每个工具连杆 × 每个料斗 Box：
       fcl::distance(tool_obj, basin_box) → 最小距离
       分类为 tool_basin
  5. 更新全局最小间隙和违规记录

碰撞判定

  • 安全阈值：clearance_threshold = 2 mm
  • 违规判定：最小距离 < 阈值
  • 优先级排序：tool_basin > tool_body > self（一个配置可能触发多种碰撞，取最高优先级）

优化策略

FCL 对象缓存

使用 thread_local 存储 FCL CollisionObject 向量。首次调用时创建所有对象；后续调用仅更新变换（setTransform）并重新计算 AABB。几何体通过 shared_ptr 共享，避免每次调用重新构建 BVH 树。

效果：单次碰撞检测从 ~47 ms 降至 ~15 ms（约 3× 加速）。

延迟碰撞验证

在 IK 求解的多种子调度中，仅对已满足位置/姿态误差容限的候选配置进行碰撞验证，避免对无效配置浪费碰撞检测时间。

稀疏审计

回放轨迹的碰撞审计采用稀疏采样（每 10 个回放点检测一次），因为相邻回放点之间的关节步长仅 ~0.7°，碰撞状态不会突变。

邻接跳过

自碰撞检测中跳过相邻连杆对（|chain_index 差 ≤ 1），因为相邻连杆在运动学上不可能碰撞。

碰撞结果结构

CollisionSummary {
  min_self, min_self_object           // 自碰撞最小间隙及对象
  min_tool_body, min_tool_body_object // 工具-本体最小间隙
  min_tool_basin, min_tool_basin_object // 工具-料斗最小间隙
  violation_type, violation_object     // 最高优先级违规
  violation_clearance                  // 违规处的间隙值
}
