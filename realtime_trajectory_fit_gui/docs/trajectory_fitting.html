轨迹拟合 & C++ 加速

问题定义

给定 N 个目标位姿 (4×4 齐次变换矩阵)，求解 UR10 机器人从初始关节角出发、依次到达每个目标位姿的关节空间轨迹，要求：

  1. 位置误差 ≤ 3 cm
  2. 姿态误差在段特定的容限内
  3. 无碰撞（自碰撞、工具-本体、工具-料斗）
  4. 关节角连续无跳变（最大回放步长 < 3°）

阻尼最小二乘法 (DLS IK)

核心求解器使用阻尼最小二乘法（Levenberg-Marquardt 风格），迭代公式：

J = ∂f/∂q (6×6 数值雅可比，中心差分，ε=1e-6)
W = diag(w₁...w₆) (任务空间权重矩阵)
H = (WJ)ᵀ(WJ) + λI (阻尼 Hessian)
g = (WJ)ᵀ(We) (加权梯度)
dq = H⁻¹g (LDLT 分解求解)
q ← clamp(align(q + dq, q_ref), limits)

自适应阻尼 λ

  • 远离目标 (pos_err > 0.1 或 rot_err > 0.1)：λ = max(1e-3, 5e-3 × pos_err/0.05)，上限 0.1
  • 靠近目标：λ = 5e-4 + 5e-3 × pos_err/0.05
  • 大步长保护：||dq|| > 0.45 rad 时缩放至 0.45 rad
  • 停滞检测：连续 8 次迭代误差下降不足 1% 时提前退出

多种子渐进式调度

对每个目标位姿，按优先级依次尝试不同种子配置：

  1. Stage 1: q_prev（上一帧解） — 利用轨迹连续性，成功概率最高
  2. Stage 2: home_q, zero — 标准参考构型
  3. Stage 3: ±2π 包裹种子 — 对腕关节 (joint 5, 6) 做 ±2π 包裹，探索等效构型分支
  4. Stage 4: 97 个全局随机种子 — 最后手段，均匀覆盖整个关节空间

每个阶段内遍历全部 4 组权重调度。一旦找到无碰撞的可行解，立即终止。

延迟碰撞验证

碰撞检测（FCL 距离查询）每次约 15-20 ms。为避免对每个种子都做碰撞检测：

  1. IK 求解器仅检查位置/姿态误差是否在容限内
  2. 仅对 IK 可行的候选调用 evaluateConfiguration 进行碰撞验证
  3. 碰撞检查通过后才记录为有效解

这使每姿态的碰撞检测次数从 ~100+ 降至 ~1-3 次。

回放轨迹生成

锚点之间使用五次多项式混合（quintic blend）：

a(t) = 10t³ - 15t⁴ + 6t⁵ (t ∈ [0,1])
q(t) = q_a + a(t) · wrapJointDelta(q_b - q_a)

子步数由关节步长（≤0.7°）、位置步长（≤3mm）、姿态步长（≤0.3°）三者共同决定，范围 [4, 32]。

关节连续性保证

  • 关节限位：每步迭代后将关节角夹紧至 [lower, upper]
  • 等效构型对齐：对每个转动关节尝试 ±2π 包裹，选择最接近参考构型 (q_prev + dq_prev) 的值
  • 包裹增量：joint delta 自动包裹至 [-π, π] 以正确处理连续关节
