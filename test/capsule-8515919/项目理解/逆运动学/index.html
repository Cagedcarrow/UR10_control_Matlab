逆运动学 (Inverse Kinematics)

给定末端 TCP 目标位姿 T，计算满足该位姿的 6 个关节角度 q，并验证路径无碰撞

1. 整体介绍

1.1 在机器人系统中的作用

逆运动学将笛卡尔工作空间的任务目标（"把铲斗移动到这个位置"）转换为关节空间的执行指令（"J1转30°, J2转-45°, ..."）。它是轨迹规划和运动控制之间的关键桥梁。

  • 轨迹规划可达性：判断给定的 TCP 位姿是否在机械臂工作空间内
  • MoveIt2 集成：MoveIt2 的规划管道依赖 IK 将笛卡尔路径转换为关节轨迹
  • 铲取轨迹执行：将 YOZ 平面规划的 3D 铲取路径转换为可执行的关节角度序列
  • GUI 交互：用户拖拽 TCP 目标时，实时求解对应关节角度

1.3 数学意义

逆运动学在数学上是一个欠定非线性优化问题：

$$q^* = \arg\min_q \underbrace{\|p(q) - p_{\text{target}}\|^2}_{\text{位置误差}} + \lambda \cdot \underbrace{\|\log(R(q)^T R_{\text{target}})\|_F^2}_{\text{方向误差}}$$

其中 $p(q) = \text{tform2trvec}(T(q))$ 是 FK 得到的 TCP 位置，$R(q)$ 是 FK 得到的旋转矩阵。

UR10 有 6 个自由度，目标位姿在 SE(3) 中有 6 个独立参数（3 位置 + 3 姿态），因此原则上解是有限多解的（通常 8 组）。本项目的求解器通过种子角度和渐进松弛策略来引导收敛到期望的构型。

1.5 在本项目中的实际用途

2.2 函数详解

solveTcpPoseIk — 核心四阶段 IK 求解器

这是整个 IK 系统的核心。采用四阶段渐进放宽策略处理不可行位姿：

function [qsol, solInfo] = solveTcpPoseIk(ikSolver, targetTform, qSeeds)
    weightSchedule = {
        [1 1 1 0.20 0.20 0.20],  30,   'strict';         % 阶段1
        [1 1 1 0.10 0.10 0.10],  45,   'relaxed';        % 阶段2
        [1 1 1 0.03 0.03 0.03],  70,   'very_relaxed';   % 阶段3
        [1 1 1 0.00 0.00 0.00],  inf,  'position_only'   % 阶段4
    };
    for stage = 1:4
        ikSolver.Weights = weightSchedule{stage, 1};
        angleLimit = weightSchedule{stage, 2};
        for each seed in qSeeds
            [qsol, solInfo] = ikSolver('sensor_shovel_tcp', targetTform, seed);
            if solInfo.ExitFlag > 0
                posErr = evaluatePosError(qsol, targetTform);
                orientErr = evaluateOrientError(qsol, targetTform);
                if posErr < 0.03 && orientErr < angleLimit
                    return;  % 成功
                end
            end
        end
    end
    error('目标不可达');
end
阶段位置权重姿态权重方向容差含义
strict[1, 1, 1][0.20, 0.20, 0.20]30°严格要求位置和姿态同时满足
relaxed[1, 1, 1][0.10, 0.10, 0.10]45°姿态权重减半，允许更大方向误差
very_relaxed[1, 1, 1][0.03, 0.03, 0.03]70°姿态权重降到 3%，基本只关心位置
position_only[1, 1, 1][0, 0, 0]∞仅位置约束，放弃方向要求

buildIkSeedList — 种子生成

function seeds = buildIkSeedList(primarySeed, homeSeed)
    seeds = {primarySeed};                          % 种子1: 当前角度
    if norm(primarySeed - homeSeed) > 0.01
        seeds{end+1} = homeSeed;                   % 种子2: 零位 (不同时)
    end
    seeds{end+1} = zeros(6, 1);                    % 种子3: 全零
end
种子策略: 三个种子覆盖了典型的收敛情形。当前角度种子适合连续性轨迹跟踪（相邻点之间关节变化小），零位种子适合从初始姿态求解，全零种子作为最后的兜底尝试。

planRRTPath — 可达性路径规划

命名澄清: 尽管函数名为 planRRTPath，其实现实际上是关节空间的线性插值(LERP)，而非标准的 RRT (Rapidly-exploring Random Tree) 算法。它通过在每个插值步长检查碰撞来验证可达性。 
function qPath = planRRTPath(q0, q1, robot, world, exemptPairs)
    stepSize = 0.03;  % 弧度
    dist = norm(q1 - q0);
    nSteps = max(2, ceil(dist / stepSize));
    qPath = interp1([0;1], [q0; q1], linspace(0,1,nSteps)');
    % 在每个中间点检查碰撞
    for k = 2:nSteps-1
        [inCollision, sepDist] = checkCollision(robot, qPath(k,:), world, ...
            'IgnoreSelfCollision', 'off', 'SkippedSelfCollisions', 'parent');
        % 过滤豁免碰撞对
    end
end

2.4 轨迹 IK 工作流

完整的铲取轨迹 IK 求解流程：

  1. 轨迹生成：generate_trajectory_yoz.m 在 YOZ 平面生成 2D 铲取路径（直线切入 → 圆弧装载 → 垂直抬升）
  2. 3D 提升：generate_trajectory_3d.m 将 2D 路径扩展到指定 X 截面（固定 $x = x_{\text{plane}}$）
  3. 构建 SE(3) 目标：buildTrajectoryTargetPlan 将 3D 位置点 + 默认朝向编码为 $4 \times 4$ 齐次变换
  4. 降采样：rtfg_downsample_target_plan_keep_boundaries 将轨迹点降至最多 140 个，同时保留段边界
  5. 逐点 IK：trackTrajectory 对每个采样点调用 solveTcpPoseIk，前一点解作为下一点的种子
  6. 关节空间后处理：可选地调用 rtfg_quintic_joint_series 对关节路径进行五次样条平滑

4. 创新点分析

4.1 与标准 IK 求解器的对比

方面标准 IK (单次调用)本项目四阶段松弛方法
不可行位姿处理直接返回失败（ExitFlag ≤ 0）逐步放宽姿态约束，提供最佳可行近似（position_only 模式保证位置收敛）
权重调优需用户手动设置单组权重自动尝试四组权重，无需人工介入
多解管理仅返回第一个局部最优解通过 3 个种子点探索不同的局部最优，选择满足容差的最佳解
轨迹连续性每点独立求解，可能产生跳变trackTrajectory 使用滑动种子确保帧间连续性

4.2 工程亮点

  • 渐进松弛策略：从严格遵守 6-DOF 约束开始，逐步放宽到仅 3-DOF 位置约束。这在实际工程中非常实用——当铲斗无法以精确姿态到达目标时，至少可以保证位置到达
  • 多种子 + 早停：每个阶段对所有种子尝试求解，一旦找到满足容差的解立即返回。避免了不必要的计算
  • 固定 30mm 位置阈值：3cm 的位置容差基于铲取任务的实际需求设定（铲斗需要在泥面附近精确定位），而非随意选择
  • 轨迹格式塔平滑：rtfg_quintic_joint_series 使用 $10t^3 - 15t^4 + 6t^5$ 五次多项式，保证位置、速度、加速度在起点和终点都为零，实现无冲击运动

4.3 局限性

  • 不是真正的 RRT：planRRTPath 仅做线性插值 + 碰撞检测，遇到障碍物时没有绕行重规划能力。如果需要避障路径规划，应替换为真正的 RRT/RRT* 或集成 OMPL
  • 纯数值方法：依赖 MATLAB 的 inverseKinematics 函数，这是一个数值迭代求解器。对于 UR10 这种有解析 IK 解的机器人，解析方法更快且能返回所有解
  • 姿态表示局限：使用权重法处理姿态约束，而非更精确的流形优化（如直接在 SO(3) 上优化）
  • 无冗余利用：UR10 在 6-DOF 任务中通常有 8 组 IK 解，本项目仅通过 3 个种子探索其中一部分。如果需要全局最优构型（如最小化关节力矩），需要更系统的搜索

4.4 未来可改进方向

  • 集成 OMPL/C++ 规划器用于含障碍物的复杂场景路径规划
  • 添加解析 IK 解（基于 UR10 DH 参数），作为数值求解器的初始化或验证
  • 在四阶段松弛中引入基于任务的自适应容差（例如，铲泥阶段需要严格姿态，空载返回阶段可以容忍更大误差）
