三维铲泥轨迹 GUI 总结文档

本文档总结 UR10_control_Matlab/test/tarjectory_plan/trajectory_plan_3d_gui 目录下 3D GUI 的构建方式、轨迹几何逻辑、参数定义、YAML 初值机制，以及 2D/3D 轨迹共享关系。

YOZ 截面轨迹 3D 可视化 URDF 环境加载 YAML 初值与保存

1. 功能概览

该 3D GUI 的目标，是在真实环境模型 block_with_basin.urdf 中观察铲泥轨迹， 并通过参数滑条快速调整轨迹的空间位置与轨迹形状。它不是机械臂逆解或控制器，而是一个 面向轨迹设计与核对的三维可视化工具。

核心思路

先在 YOZ 平面 上生成一条二维铲泥轨迹，再把这条二维轨迹整体抬升到 三维空间中的某一个固定 X = xPlane 截面内。

共享几何层

二维 GUI 与三维 GUI 共用同一个核心轨迹生成器 generate_trajectory_yoz.m， 从而保证 2D 和 3D 的轨迹形状严格一致。

三维专属能力

三维 GUI 额外提供 xPlane 截面位置控制、X 向两侧壁距离显示、环境 URDF 加载、 三维视角切换、姿态箭头和世界坐标系显示。

2. 文件构成与职责

文件 作用
main_gui_3d.m 3D GUI 入口。负责窗口布局、滑条、按钮、YAML 读写入口、回调绑定、刷新驱动。
build_gui_state.m 构建默认状态、环境几何参数、盆体内腔范围、YAML 初值加载与参数限幅。
generate_trajectory_yoz.m 原生二维轨迹生成器。直接在 YOZ 截面中构造“预运动 + 泥下圆弧 + 竖直提升”轨迹。
generate_trajectory_3d.m 将二维 YOZ 轨迹抬升到固定 X 平面，形成三维轨迹。
render_scene.m 统一负责 3D 场景渲染：URDF 环境、轨迹点线、关键点、泥面、截面辅助面、坐标系、箭头。
trajectory_params_3d.yaml 3D GUI 的启动初值文件，同时也是“导出参数”和“保存当前为初始值”的落盘目标。

3. GUI 构建流程

  1. 调用 build_gui_state('trajectory_params_3d.yaml') 构建状态。
  2. 如果同目录存在 trajectory_params_3d.yaml，则读取其中 parameters 段作为初始值。
  3. 加载环境模型 environmental_model/block_with_basin.urdf。
  4. 创建左侧参数区与右侧三维场景区。
  5. 绑定参数滑条、视角下拉框、显示开关、导出按钮、保存初值按钮。
  6. 根据当前参数调用 generate_trajectory_3d 生成轨迹，再调用 render_scene 渲染。
当前 3D GUI 在启动时不是固定读代码默认值，而是优先读 trajectory_params_3d.yaml。 因此只要在 GUI 里点击“保存当前为初始值”，下次打开窗口时就会自动从该 YAML 恢复这组参数。

4. 盆体与坐标定义

盆体几何在 build_gui_state.m 中定义，关键量如下：

  • basin.worldOrigin = [-0.135, 0.0, 0.25]：盆体内腔参考原点。
  • outerLengthX = 0.37 m：盆体外部长。
  • outerWidthY = 0.50 m：盆体外部宽。
  • outerHeightZ = 0.18 m：盆体高度。
  • wallThickness = 0.003 m：壁厚。
  • 由此推导出内腔尺寸、底面高度 floorZ、盆沿高度 rimZ、以及 xPlane 允许范围。

该 GUI 使用的空间理解是：

  • X 方向：盆体长度方向，也是选择不同 YOZ 截面的方向。
  • Y 方向：YOZ 截面中的横向方向，用于控制入泥点距左侧壁距离。
  • Z 方向：竖直方向，盆底到盆沿，以及泥面高度都定义在这个方向。

5. 轨迹分段逻辑

轨迹由 3 段构成，全部先在 YOZ 平面内生成：

  1. 预运动段 seg0：从桶外斜上方的起始运动点直线移动到入泥点。
  2. 圆弧装载段 seg1：从入泥点开始，在泥面以下沿圆弧旋转，直到末端方向与泥面平行。
  3. 竖直提升段 seg2：圆弧结束后，沿全局 +Z 方向直接提升。

5.1 入泥点

入泥点始终位于泥面上，因此其二维坐标不是自由设定的两个量，而是： 

yEntry = basin.innerYMin + leftWallOffset
zMud   = basin.floorZ + mudHeight
pEntry = [yEntry, zMud]

这里 leftWallOffset 控制入泥点在 Y 方向的位置，mudHeight 控制泥面高度。

5.2 预运动起始点

起始运动点不是单独输入，而是由入泥点、入泥角、斜线长度自动反推： 

theta    = deg2rad(thetaDeg)
dirEntry = [cos(theta), sin(theta)]
pApproachStart = pEntry - approachLen * dirEntry

这里 approachLen 表示从起始运动点到入泥点的直线长度。由于 thetaDeg 为负， 所以这条线会从入泥点往左上方回推，形成桶外斜上方的起始运动点。

5.3 圆弧段的核心定义

圆弧段的几何目标是：

  • 从入泥角 thetaDeg 开始。
  • 圆弧结束时，轨迹方向与泥面平行。
  • 参数 depth 表示的是相对泥面的实际垂直下扎深度，不是圆弧半径。

因此代码中首先反算圆弧半径：

depthScale = 1 - cos(abs(theta))
radius     = depth / depthScale

这个公式来自圆弧终点相对泥面的竖向下降量。对于当前几何定义，圆弧结束点相对泥面的下沉量是： 

verticalPenetration = radius * (1 - cos(abs(theta)))

令这个量等于用户输入的 depth，即可得到上面的反算半径公式。

圆弧的起始角与结束角定义为：

a1 = theta - pi / 2
a2 = -pi / 2

对应圆心坐标为：

center = pEntry - radius * [cos(a1), sin(a1)]

圆弧上的点按参数角 a 插值得到：

seg1(a) = [
    center(1) + radius * cos(a),
    center(2) + radius * sin(a)
]

其中 a 从 a1 连续变化到 a2。圆弧终点即为：

pArcEnd = seg1(end, :)
这也是为什么当前 GUI 中如果 theta 绝对值很小、但 depth 设得很大，反算出来的圆弧半径会非常大。 这是几何关系导致的，而不是比例错误。

5.4 竖直提升段

圆弧结束后，不再沿切线方向抬起，而是直接沿世界坐标 +Z 方向竖直提升。代码采用： 

liftLen = min(basin.zMotionMax - pArcEnd(2), max(0.06, 1.20 * radius))
seg2 = [repmat(pArcEnd(1), nLift, 1), zLift]

这里提升长度既受盆体上边界限制，也保留了一个与圆弧半径相关的默认提升长度策略。

6. 二维到三维的映射

轨迹先在 YOZ 平面中生成，再由 generate_trajectory_3d.m 抬升到三维空间。映射公式非常直接： 

[Y, Z] in YOZ plane  --lift-->  [X, Y, Z] in world

X = xPlane
Y = original 2D Y
Z = original 2D Z

具体实现为：

pts3 = [repmat(xPlane, size(pts2, 1), 1), pts2(:, 1), pts2(:, 2)]

因此：

  • 修改 xPlane 只会让轨迹在 X 方向平移。
  • 不会改变 YOZ 截面内的轨迹形状。
  • 二维 GUI 里看到的轨迹，就是三维 GUI 某个固定截面的真实截面轨迹。

7. 参数含义

参数 变量名 含义
距左侧壁距离 leftWallOffset 入泥点到盆内左侧壁的水平距离，决定入泥点在 Y 方向的位置。
泥面高度 mudHeight 泥面相对盆底的高度，决定入泥点和泥面的 Z 位置。
起始运动斜线长度 approachLen 从起始运动点到入泥点的直线长度，用于控制桶外预运动段长度。
入泥角度 thetaDeg 预运动段与圆弧起始切向的角度，负值表示向下切入。
入泥深度 depth 圆弧终点相对泥面的实际垂直下扎深度，不是圆弧半径。
轨迹平面 X 位置 xPlane 三维轨迹所在的固定 YOZ 截面位置。

8. GUI 交互说明

  • 滑条：用于实时调整 6 个核心参数。
  • 距 X-/X+ 侧壁距离：只读显示，用于观察当前 xPlane 离两侧壁的距离。
  • 视角下拉框：可切换 Isometric / Top / Front / Side。
  • 显示世界坐标系：控制 X/Y/Z 坐标箭头显示。
  • 显示姿态箭头：控制沿轨迹切向方向的箭头显示。
  • 导出参数为 YAML：将当前参数和推导值导出到 trajectory_params_3d.yaml。
  • 保存当前为初始值：将当前参数覆盖写入 trajectory_params_3d.yaml，下次启动自动加载。

9. YAML 初值与保存机制

3D GUI 启动时，会在 build_gui_state.m 中读取同目录的 trajectory_params_3d.yaml。 当前只读取其中 parameters 段里的以下字段： 

left_wall_offset
mud_height
approach_len
theta_deg
depth
x_plane

如果 YAML 中的参数超出 GUI 允许范围，代码会先进行限幅，然后再作为 GUI 初值使用。 对应的限幅范围目前写在 build_gui_state.m 的 clampParamsToGuiLimits 中。

当前 3D YAML 文件里除了 parameters，还会导出以下推导量：

  • approach_start_point_xyz
  • entry_point_xyz
  • arc_end_point_xyz
  • approach_length
  • arc_radius
  • vertical_penetration
  • dist_x_negative_wall
  • dist_x_positive_wall

10. 渲染策略

场景渲染在 render_scene.m 中统一完成。为了减少 MATLAB UIAxes/axes 频繁重绘带来的问题， 当前采用的是稳定优先策略：

  • 每次刷新先 cla(ax)。
  • 用 show(..., 'PreservePlot', false, 'FastUpdate', true) 绘制环境模型。
  • 再叠加轨迹、关键点、泥面辅助面、截面辅助面、姿态箭头与世界坐标系。
  • 相机状态在 GUI 层单独保存并恢复，从而避免拖动滑条时重置视角。

11. 典型使用流程

  1. 运行 main_gui_3d。
  2. GUI 自动从 trajectory_params_3d.yaml 读入上次保存的参数。
  3. 通过滑条调整入泥点位置、泥面高度、预运动长度、入泥角、垂直入泥深度以及截面位置。
  4. 在三维视图中确认轨迹是否位于盆体内部、是否在泥面以下完成圆弧转平。
  5. 如果需要对外共享当前配置，点击“导出参数为 YAML”。
  6. 如果需要把当前配置作为下次启动默认值，点击“保存当前为初始值”。

12. 当前实现特点与注意事项

  • 当前 3D GUI 的轨迹几何与 2D GUI 共用一套核心逻辑，因此二维核对与三维显示是一致的。
  • depth 的定义已经明确为垂直入泥深度，不能再按“圆弧半径”理解。
  • 如果 theta 太接近 0 度而 depth 又较大，圆弧半径会很大，这是几何必然结果。
  • 当前文档描述的是轨迹生成与可视化逻辑，不包含机械臂逆运动学、关节规划、碰撞求解或控制发送。
